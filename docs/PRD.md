# 产品需求文档 (PRD) - 自动记账App (MoneyTracker)

## 1. 产品概述 (Product Overview)

**产品名称：** MoneyTracker (暂定)

**产品定位：** 一款专为 Android 用户设计的、以“自动化”和“个人净资产管理”为核心的纯本地开源记账应用。

**目标用户：**

- 厌倦了繁琐手动记账的用户。
- 拥有多张信用卡/花呗，需要管理净资产和债务的用户。
- 注重数据隐私，排斥商业记账软件中充斥的广告和社区功能的用户。

**核心痛点解决：**

1. **“记账难坚持”**：通过屏幕和通知监听，实现高比例的自动记账弹窗。
2. **“负债算不清账”**：引入资产负债双账户模型与转账逻辑，准确计算净资产。
3. **“隐私担忧”**：无服务端账户体系，纯本地 SQLite (Room) 存储。

## 2. 核心架构与业务逻辑 (Core Architecture & Logic)

### 2.1 财务模型 (Financial Model)

本应用采用“复式记账法”的简化版，支持精确计算净资产。

- **账户分类：** 分为`资产 (ASSET)` 和 `负债 (LIABILITY)`。
- **净资产公式：** 净资产总计 = 资产账户总额 - |负债账户总额|
- **核心交易类型：**
  - `支出 (EXPENSE)`：资产减少 或 负债增加（如：用微信余额点外卖、用花呗购物）。
  - `收入 (INCOME)`：资产增加 或 负债减少（如：发工资、收到转账）。
  - `转账 (TRANSFER)`：资产间的转移，或用资产偿还负债（如：从招行卡还花呗，此时净资产不发生变化）。

### 2.2 自动化抓取引擎 (Automation Engine)

自动化引擎是本产品的核心竞争力，采用**双层防御策略**：

1. **主动出击层 (AccessibilityService)**
   - **触发条件：** 用户进行扫码、付款码支付等主动操作。
   - **作用：** 监听主流支付 App（微信、支付宝）的“支付成功”或“账单详情”特定页面节点。
   - **提取物：** 金额、商户名/对方昵称、支付方式。
2. **被动防御层 (NotificationListenerService)**
   - **触发条件：** 发生免密支付、自动续费、先用后付扣款等无支付界面场景。
   - **作用：** 监听系统状态栏中来自银行、微信、支付宝的扣款通知文本。
   - **提取物：** 配合正则表达式提取金额、交易对象。
3. **防重机制 (De-duplication)**
   - 针对同一笔交易可能同时触发上述两层的场景，生成 `rawHash` (基于时间戳取整+金额+部分商户名)，并在写入数据库前进行查重。

## 3. 功能模块详细说明 (Feature Specifications)

### 3.1 记账与账单管理 (Transaction Management)

#### 3.1.1 自动记账弹窗 (核心交互)

- **场景：** 自动化引擎捕获到交易数据。
- **行为：** 使用 `WindowManager` 在当前屏幕上方弹出一个半透明悬浮卡片。
- **卡片元素：**
  - 自动填入的金额与预判的账户。
  - 快速选择分类的网格视图（根据历史记录预测或展示常用分类）。
  - [确认保存] 按钮。
  - [关闭/忽略] 按钮。
  - [展开详情] 按钮（点击进入全屏手动修改页面）。

#### 3.1.2 手动记账

- **入口：** 首页底部的 [+] 悬浮按钮 (FAB)。
- **支持类型：** 支出、收入、转账。
- **必填项：** 金额、账户。
- **选填项：** 分类（转账无需分类）、日期（默认当前时间）、备注。
- **交互：** 内置计算器键盘。

#### 3.1.3 退款标记 (Refund System)

- **场景：** 用户在账单详情中点击一笔历史支出，选择“退款”。
- **逻辑：** * 支持输入退款金额（全额或部分）。
  - 原账单状态变更为 `REFUNDED` 或 `PARTIAL_REFUNDED`。
  - 自动生成一笔新的 `INCOME` 类型账单，金额为退款额，入账账户同原账户，`originalId` 关联原账单。

### 3.2 账户与分类管理 (Account & Category)

#### 3.2.1 账户管理

- 支持创建无限量账户。
- 创建时必须选择类型（资产/负债）。
- 列表支持拖拽排序。
- 支持设置账户“隐藏”（不计入首页总资产）。

#### 3.2.2 树状分类管理

- **层级：** 支持父子关系（理论无限级，建议 UI 展示限制为 2-3 级）。
- **内置数据：** App 首次启动时，提供一套基础实用的分类模板。
- **自定义：** 支持添加、删除、重命名、更换图标。删除父分类时，连带删除或转移子分类。

### 3.3 数据可视化与报表 (Visualization & Reports)

#### 3.3.1 资产大盘 (首页/Dashboard)

- 顶部显著位置展示：当前净资产、本月总支出、本月总收入。
- 横向滑动卡片展示各个主要账户的当前余额。

#### 3.3.2 日历视图 (Calendar View)

- 在日历的每一天上，以小字显示当天的“总支出”和“总收入”（或以不同颜色的小圆点标记）。
- 点击某一天，在下方展开该天的详细账单列表。

#### 3.3.3 统计报表 (Charts)

- **饼图/环状图：** 展示选定时间段（月/年）内，各项支出/收入分类的占比情况。
- **柱状/折线图：** 展示近 6 个月或 1 年内的收支趋势。

### 3.4 数据安全与流转 (Data I/O)

- **完全离线：** 不请求任何网络权限（除了可选的 AI 模块和应用内版本更新检查）。
- **账单导入：** 支持解析标准的支付宝 CSV 账单和微信 Excel 账单。根据时间戳、金额进行智能防重导入。
- **账单导出：** 支持将数据库中的所有记录导出为标准 CSV/Excel 格式或 JSON 格式，方便用户在电脑端二次处理或备份。

### 3.5 AI 智能助手 (AI Assistant - 附加/进阶功能)

- **配置界面：** 提供独立的设置页，允许用户输入大模型服务商配置信息（Base URL, Model Name, API Key）。
- **财务诊断功能：**
  - 在报表页提供一个 [AI 分析] 按钮。
  - 将当前月份的汇总数据（脱敏 JSON 格式：仅包含分类、金额、时间、不包含具体商家和隐私信息）发送给大模型。
  - 展示大模型生成的“消费结构评价”、“不合理开销预警”和“省钱建议”。

## 4. 技术栈规范 (Technology Stack Constraints)

- **语言:** Kotlin (100%)
- **UI 框架:** Jetpack Compose (禁止使用传统 XML 布局)
- **架构模式:** MVI (Model-View-Intent) 或 MVVM 配合 Kotlin Coroutines & Flow
- **本地数据库:** Room Database (已有基础 Entity 设计)
- **依赖注入:** Hilt / Dagger
- **网络请求 (仅用于 AI 模块):** Retrofit2 + OkHttp3
- **JSON 解析:** Kotlinx Serialization 或 Moshi
